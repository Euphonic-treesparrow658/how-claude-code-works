## 🧩 Key Ideas Explained Simply

### Agent Loop

The agent loop is the repeat cycle that Claude Code follows. It takes input, thinks about the next step, picks a tool, and checks the result. Then it repeats.

### Context Engineering

Context engineering is how the app keeps the right information in view. It decides what to include, what to trim, and what to send next. This helps the agent stay on task.

### Tool System

The tool system lets the agent act on files, run commands, or gather data. Each tool has a clear job. The app sends a request, gets a result, then uses that result in the next step.

### Architecture

Architecture is the way the whole app is put together. It shows how the parts connect, how messages move, and how state changes during use.

## 🔄 How the Project Is Organized

A project like this often uses a simple flow:

1. Input comes in from the user.
2. The agent reads the input.
3. The app checks the context it has.
4. It chooses a tool if needed.
5. The tool returns a result.
6. The agent updates its state.
7. The loop continues until the task ends

That flow is the core idea behind Claude Code behavior.
